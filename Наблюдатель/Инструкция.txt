Ты senior Python developer и data engineer. Нужно создать **готовый парсер каталога HiWatch** с сайта **https://hiwatch.net/**, который **обойдет все категории и все карточки товаров**, соберет **полный список товаров без цен** и сохранит данные в **структурированной файловой системе**.

## Цель

Нужно получить локальную базу товаров HiWatch со следующими данными по **каждой позиции**:

* название товара
* модель
* категория / подкатегория
* основное фото товара
* описание товара
* технические характеристики
* паспорт / datasheet / инструкция / pdf-документ, если он есть на странице
* ссылка на страницу товара
* артикул / серия / сопутствующие поля, если они есть на сайте

## Важно

* **Цены собирать не нужно**
* Нужно пройтись **по всем вкладкам / категориям / подкатегориям**
* Нужно собрать **все товары сайта**, а не только первую страницу
* Описание и технические характеристики нужно сохранить в **XML**
* Для каждого товара должна создаваться **отдельная папка**
* Парсер должен быть устойчивым, аккуратным и пригодным для реального использования

---

## Требуемая структура папок

Нужно сохранить всё в таком виде:

```text
HiWatch/
├── IP камеры/
│   ├── DS-I200/
│   │   ├── image.jpg
│   │   ├── passport.pdf
│   │   └── product.xml
│   ├── DS-I400/
│   │   ├── image.jpg
│   │   ├── passport.pdf
│   │   └── product.xml
│
├── Видеорегистраторы/
│   ├── DS-N204/
│   │   ├── image.jpg
│   │   ├── passport.pdf
│   │   └── product.xml
│
└── ...
```

Если у товара несколько фото:

* либо сохраняй главное фото как `image.jpg`
* а дополнительные как `image_2.jpg`, `image_3.jpg`

Если паспорта нет:

* просто не создавай pdf-файл
* но в XML укажи, что документ отсутствует

---

## Что должно быть внутри XML

Для каждого товара нужно создать файл `product.xml` в понятной и чистой структуре.

Пример желаемой структуры XML:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<product>
    <name>IP-камера HiWatch DS-I200</name>
    <model>DS-I200</model>
    <category>IP камеры</category>
    <subcategory></subcategory>
    <product_url>https://hiwatch.net/...</product_url>

    <description>
        <paragraph>Текст описания 1</paragraph>
        <paragraph>Текст описания 2</paragraph>
    </description>

    <specifications>
        <spec>
            <name>Разрешение</name>
            <value>2 Мп</value>
        </spec>
        <spec>
            <name>Объектив</name>
            <value>2.8 мм</value>
        </spec>
        <spec>
            <name>Подсветка</name>
            <value>до 30 м</value>
        </spec>
    </specifications>

    <files>
        <image>image.jpg</image>
        <passport>passport.pdf</passport>
    </files>
</product>
```

## Логика парсинга

Нужно реализовать полный обход сайта:

1. Зайти на сайт
2. Найти все товарные категории
3. В каждой категории найти все товары
4. Открыть каждую карточку товара
5. Вытащить:

   * название
   * модель
   * категорию
   * описание
   * технические характеристики
   * фото
   * ссылку на паспорт / datasheet / pdf, если есть
6. Скачать файлы
7. Сформировать XML
8. Разложить всё по папкам

## Технические требования к реализации

Сделай проект на **Python**.

Предпочтительно:

* `requests`
* `BeautifulSoup4`
* `lxml`
* `pathlib`
* `concurrent.futures` или `asyncio/aiohttp` только если это реально уместно
* `logging`

Если сайт сильно зависит от JavaScript:

* используй `Playwright`
* но сначала попробуй обычный HTML-парсинг
* если данных в исходном HTML нет, тогда переходи на browser automation
